## Design Notes

**Interface subversion as the core mechanic.** Pony Island's central design insight is that players bring strong assumptions to every game interface — menus mean safety, error screens mean something broke, code means the game is broken — and that these assumptions can be violated for horror effect. When Pony Island's title screen glitches, players lean forward. When the game's options menu is locked behind a puzzle, players feel they've found something hidden. When Lucifer addresses them directly through what appears to be a system error, it produces genuine unease. Mullins didn't invent this technique but codified it into a complete game: the medium is the horror, not the content.

**Minimizing instruction as mystery.** Mullins designed Pony Island with almost no explicit tutorial or instruction, not because tutorials are bad but because being told nothing is consistent with the fiction. If you discovered a corrupted arcade machine, it wouldn't have a help menu. The lack of instruction means players arrive at each discovery with the feeling that they weren't supposed to find it, which produces a qualitatively different engagement than designed discovery sequences. The game feels like uncovering a secret rather than following a plot. This is a design technique that scales poorly but that Mullins deployed with precision: the game is short enough that disorientation never becomes frustration.

**Layered narrative with distinct mechanical grammars.** Pony Island operates simultaneously at three narrative levels: the surface game (a side-scrolling pony arcade game), the corrupted system underneath it (code puzzles and hacking sequences), and the meta-layer where Lucifer communicates directly with the player. Each layer uses different mechanics to communicate its fictional status. The arcade game uses typical platformer logic. The hacking sequences use pseudocode that looks threatening but follows learnable rules. Lucifer's addresses break the fourth wall entirely. The design achievement is that players always know which layer they're in without being told, because the mechanics themselves carry that information.

**Pseudocode as horror element.** The code puzzles in Pony Island use simplified pseudocode that players manipulate to unlock game files or defeat Lucifer's corruptions. The code is designed to read as ominous — variable names that suggest soul ownership, function calls that feel irreversible — while remaining functionally solveable. This is a rare example of designing programming challenge content for emotional effect rather than educational clarity. The puzzles work because they produce the feeling of doing something you shouldn't know how to do, while the actual solution space remains accessible. It's horror through aesthetic, not through difficulty.

**Social anxiety as design material.** Pony Island includes a sequence that causes players to believe they have sent an insulting message to a friend from their Steam account. The sequence is fake — no message is sent — but the panic it produces is real and measurable. Mullins used this to demonstrate that the most effective horror in games isn't threat to the player character but threat to the player's social identity and real-world relationships. This technique has a high blast radius — players who understand what happened feel the anxiety retrospectively, not just during the moment — and requires careful placement in the game's structure. It's a design move that only works once and must be deployed at a moment when players have enough trust to be properly deceived.
